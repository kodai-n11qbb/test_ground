# 画像差分検出プロジェクト - 仕様書

## プロジェクト概要

originディレクトリの元データとdummyディレクトリのミス生成データの差分を検出し、視覚的に表示するシステム。

## 目的

- 正解データ（origin）と誤りデータ（dummy）の差分を特定する
- 位置が違っても形状が同じならOKと判定する（位置不変性）
- CADデータとカメラ入力の一致度合いを測定するアプローチを採用
- 精度調整が可能なパラメータを提供する

## 機能要件

### 1. 画像読み込み
- originディレクトリから元データを読み込む
- dummyディレクトリから比較対象データを読み込む
- ファイル名に含まれる「カッコ内の文字列（キーワード）」を抽出し、キーワードが一致する元画像と比較対象画像をすべてペアリングする（1対多対応）

### 2. 前処理

#### 2.1 実写の CAD 風正規化（カメラ入力）
origin（CAD 相当）と dummy（カメラ実写）の**表現域を揃え、撮影角度による歪みを補正してから**照合する。

| 課題 | 定石 | 本プロジェクトでの対応 |
|------|------|------------------------|
| 実写は木目・照明・透視を含む | 照合前にドメイン正規化 | HSV で文字色抽出 → 白背景＋紺フラット化 |
| 手ブレや斜め撮影による歪み | 射影変換（台形補正） | 青色領域から看板の4隅を検出し長方形に補正 |
| origin は既に 2 値に近い | 比較対象を同じ特徴空間へ | 補正後の画像を origin と同サイズへ変換 |
| 合成 dummy は既に CAD 風 | 不要な変換は避ける | 解像度比が閾値以下なら正規化スキップ |

**適用条件**: `max(dummy高, dummy幅) > max(origin高, origin幅) × photo_size_ratio_threshold`（既定 1.5）

**処理手順**:
1. 下部クロップ（床・足の除外、`photo_bottom_crop_ratio`）
2. 青色領域（看板の外枠など）の最大輪郭を検出し、その輪郭を4点に近似（`cv2.approxPolyDP`）して4隅（コーナー）を特定する。
3. 特定した4隅の座標を基準に、射影変換（`cv2.getPerspectiveTransform` および `cv2.warpPerspective`）を実行し、歪みのない長方形（originと同じ縦横比およびサイズ）に補正・リサイズする。
4. 補正後の画像に対して再度HSV青マスク処理を行い、白背景 + 紺塗り（originと同系色）の文字シルエット画像を生成する。

**効果（実写直近・POLITEC ドトール）**: 正規化および台形補正後は画素差分マスク面積が大幅減少、輪郭の重ね合わせ精度が向上する。

#### 2.2 形状マッチング用前処理
- グレースケール変換
- ノイズ除去（ガウシアンブラー、カーネルサイズ：7）
- リサイズ（正規化後は origin サイズ済み）

### 3. 輪郭抽出
- Cannyエッジ検出による輪郭抽出（閾値：30.0 / 100.0）
- findContoursによる輪郭座標の取得
- 複数輪郭を凸包で統合して1つの形状として扱う
- Huモーメント計算による形状特徴抽出（位置不変性）
- 両画像（originおよび台形補正後のdummy）の輪郭データを生成

### 4. 差分検出
- Huモーメントによる形状マッチング（位置不変、符号付き対数スケール化によるスケール差補正を適用）
- 2つの類似度計算方法（対数スケールHu差分ベース `diff`、凸包輪郭の `cv2.matchShapes` `matchshapes`）
- 形状類似度スコアの計算
- 閾値によるOK/NG判定（調整可能）
- 絶対差分（画素単位）を用いた物理的な差分マスク（はみ出し・欠け）の特定
- 輪郭同士の重ね合わせアライメント調整

### 5. 可視化
- 元画像とダミー画像のブレンド画像上に、**originの輪郭（緑など）と補正後dummyの輪郭（赤など）をそれぞれ重ねて描画（オーバーレイ）**した画像ファイルの出力
- 結果のJSON出力（他言語連携用）
- FastAPIとHTML/JSによるWebベースの差分ビューア

## 技術スタック

- **言語**: Python 3.x
- **画像処理**: OpenCV (cv2)
- **Web**: FastAPI, uvicorn
- **可視化**: 画像ファイル出力、Webビューア（HTML/JS/CSS）
- **テスト**: pytest

## 入出力

### 入力
- `imgs/origin/`: 元データ画像（CADデータ相当）
- `imgs/dummy/`: 比較対象画像（カメラ入力相当）

### 出力
- 差分検出結果の画像ファイル（output/）
- 結果のJSONファイル（他言語連携用）
- 類似度スコア

## 制約事項

- originとdummyは、ファイル名のカッコ内キーワードが一致する組み合わせでペアリングする（1対多可）
- 画像サイズは一致している必要がある（または自動リサイズ）
- [DEV_POLICY.md](../DEV_POLICY.md)の開発方針に従う
